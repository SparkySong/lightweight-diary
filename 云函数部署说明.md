# 云函数部署说明

## 问题概述
当前出现的错误 `RuleNotExists: rule for getProfile not exists` 是由于云函数权限规则不存在导致的。我已经为所有云函数添加了必要的 `config.json` 文件，但需要在微信开发者工具中重新部署这些云函数。

## 解决方案

### 步骤一：重新部署所有云函数
1. 在微信开发者工具中，点击左侧的 **"云开发"** 图标
2. 进入 **"云函数"** 选项卡
3. 你会看到以下云函数列表：
   - addDietRecord
   - addRecord
   - deleteDietRecord
   - deleteRecord
   - getDietRecords
   - getGoal
   - getProfile
   - getRecords
   - sendReminder
   - setGoal
   - setHeight
   - subscribeReminder

4. 对 **每个** 云函数执行以下操作：
   a. 右键点击云函数名称
   b. 选择 **"上传并部署：云端安装依赖"**
   c. 等待部署完成

### 步骤二：检查云函数权限（如果重新部署后仍有问题）
1. 在微信开发者工具的 **"云开发"** 面板中
2. 进入 **"数据库"** 选项卡
3. 确保以下集合存在：
   - `weight_records`
   - `user_profiles`
   - `weight_goals`
   - `user_reminders`
   - `diet_records` (如有需要)

### 步骤三：本地测试云函数
1. 在 `index.js` 文件中，我已经添加了容错机制：
   - 如果云函数调用失败，会尝试使用本地存储的数据
   - 页面仍然可以基本正常显示

## 重要提示

### 部署顺序：
- 建议按照以下顺序部署：
  1. getRecords, getGoal, getProfile (数据读取)
  2. addRecord, deleteRecord (数据写入)
  3. 其他次要云函数

### 如果仍然出现权限错误：
可能需要手动设置数据库权限：
1. 进入 **"数据库"** 选项卡
2. 对每个集合点击 **"权限设置"**
3. 选择 **"所有用户可读，仅创建者可写"**

### 环境检查：
确保你的云环境选择正确：
1. 查看 `app.js` 中的初始化代码：
   ```javascript
   wx.cloud.init({
     traceUser: true,
     env: wx.cloud.DYNAMIC_CURRENT_ENV
   });
   ```
2. `DYNAMIC_CURRENT_ENV` 会自动使用当前云开发环境

## 文件修改汇总
我已经完成了以下修改：

### 1. 为所有云函数添加了 `config.json` 文件：
每个 `config.json` 内容：
```json
{
  "permissions": {
    "openapi": []
  }
}
```

### 2. 改进了 `index.js` 的错误处理：
- 使用 `Promise.allSettled()` 代替 `Promise.all()`
- 添加了本地存储作为回退方案

### 3. 问题诊断：
根据错误信息，最可能的原因是：
- 云函数从未部署过，或者部署后配置未生效
- 云开发控制台的权限设置需要手动配置

执行完以上步骤后，重新编译小程序应该能解决 `RuleNotExists` 错误。
